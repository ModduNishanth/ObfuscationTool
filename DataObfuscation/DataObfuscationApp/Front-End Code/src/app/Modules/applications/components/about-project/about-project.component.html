• Data obfuscation is also known as data masking or data anonymization. This process involves transforming sensitive or personally identifiable information (PII) into a non-sensitive and anonymous form, all while retaining its usability for specific purposes. The primary objective of data obfuscation is to safeguard sensitive data from unauthorized access, reducing the risks associated with data breaches, unauthorized disclosures, or privacy violations.

  • To enhance security and confidentiality, we can utilize the Kalpita Obfuscation Tool to obfuscate the client-provided data.
  • Project - This page enables the setup of a project by providing database connection information and allows users to test the connection. Once the project is set up, all tables in the database will be available for obfuscation.
  • Mapping Tables - Once the database connection is established, the Mapping Page allows the user to select tables for obfuscation and map the obfuscation function to each column in the table that needs to be obfuscated.
  • Admin Page - This menu has two functionalities: Keyword and Function.
  • Obfuscation - This will display the mapped tables, and by selecting these tables, we can obfuscate the data for the columns that have been mapped in the client database.

On this page, we will set up a new project and provide a Server connection string for a Database connection.

Functionality of this page :
  • Adding new Project
    • Click the + icon on the page to add a new project.
  • Enter Project Name, Project Description, Server Type, Server Name, User Name, Database Name, Password and Click Save to create a New project
  • Project Name should be Unique. One can have only one Project created for a Server and Database
  • Server Type: Currently this tool is tested for SQL Server 2019, MySql 8.0 and Oracle 19
  • Modifty Project Details :
    • Click on Edit Icon to Modify the Project Details
  • Copy Attributes :
    • This feature enables the copying of columns from another project's data. This helps users replicate the mappings in another environment without having to re-do them. It copies only the mapped attributes of the selected project.
  • Test Connection :
    • Click on Test Connection Icon to validate if connection details provided is Proper.
      > Green Icon - Connected
      > Red Icon - Connection Failed
      > Orange - Ready for connection
  • If you wish to delete the project, ensure that no data is mapped to it. If any data is mapped, please delete it from the obfuscation list.

How to Process :
  • Select Project name.
  • Select Table name.
  • For the initial setup, click the "Fetch Columns" : Click "Fetch Columns" button to fetch the columns in the table. The first time you click, the "New" icon will be shown. Fetching columns will retrieve the column names into the Obfuscation Tool (not the data). Users can click "Fetch Columns" again at a later date to sync the column information, which is useful for newly added columns or deleted columns in the table. Upon subsequent clicks of "Fetch Columns," if the column is already synced, it will show the icon as "Existing." If the column has been deleted, the icon will be displayed as "Deleted."
  • Search text box is also provided for searching the columns
  • Once we have the column information, users can select the checkbox for the columns to be obfuscated and choose the appropriate obfuscation function. Selecting a function that does not support the data types of columns may lead to data misbehavior. For example, choosing a Date Obfuscation function for a string data type column.
  • User can Preview the data (top 5 data) before doing obfuscation. It's important to note that the preview feature will not work for the Certificate Function.
  • To expedite the column selection for obfuscation, users can choose to apply a keyword, which will automatically select the columns to be obfuscated along with the corresponding function based on the keyword setup in the Admin: Keyword page. For instance, setting up the keyword "date" will automatically select all columns with the name "date." Once the columns are selected for obfuscation, click "Map Columns" to save the mappings
  • Users can export the mappings to Excel, allowing them to send them for review.
    • How to Use Certificate :
  • To utilize the Certificate function, you can add a column to your table with the name "Encrypted_Value" and a data type of varbinary(max). It's important to note that the Certificate function can be applied to one column at a time.
  • If you possess multiple certificates, you need to add a column to each table with the name "Encrypted_Value" for the desired functionality. It's essential to note that for each table, only one column can be designated for certificate functionality, meaning you cannot add multiple columns at once. However, you can apply the certificate function to multiple columns sequentially, but not simultaneously.
  • Note : Ensure that your symmetric key is named "SymmetricKey1" for this functionality to work correctly.
  • After completing all column mappings, be sure to click the "Map Columns" button to finalize the process.
  • After mapping, if you need to unmap columns, simply deselect the columns you wish to unmap and then submit the changes.
  • To verify the data columns, you can export the information to an Excel sheet.
  • Note : If the columns are not mapped with the proper functionality, it may lead to data misbehavior. Ensure that the mapping is done correctly.

  • This page enables you to deploy the obfuscation functions onto your database. These functions must be present in your database for the data to be successfully obfuscated.
  • Select all the function and Click Deploy to deploy the function. User metioned in the Project Connection is required to have the Permission to create the functions. Use Show Query to Preview the defn of Function
  • Creating certificates is not supported within this application; you must create them manually on your server. However, once you have created a certificate, you can utilize it within this application. You don't have the option to deploy the Certificate Function directly from this application.
  • Note : Ensure that you deploy these functions before you commence the obfuscation process.
    List of Functions
  • Email Function: Email Function is used to mask the data by generating random alphabets with email formate.
    Example : Obfuscation@gmail.com > abcdefgh@example.com
  • RANDOM TEXT Function: RANDOM TEXT Function is used to generate random alphabets based on the length of the original data.
    Example : Obfuscation > abcdefghmopnm
  • Number Function: Number Function is used to generate the random Numbers based on length of the original data.
    Example : 9876746322
  • DATE Function: Date Function is used to mask the date. The date can be in any formate its masks the date.
    Example : 01/01/2024 > **/01/****
  • (#)TEXT Function: (#)TEXT Function is used to mask the data by generating # or * in original data.
    Example : Obfuscation > Ob*******on
  • SSN Function: SSN Function is used to mask the data by generating X in original data. This can used to mask SSN numbers, Aadhar numbers.
    Example : 111-22-3333 > XXX-XX-X333
  • Constant Value: The Constant Value function is used to apply the same value to all rows in a specific column. There is no need to deploy this function.
  • Certificate: While there is no specific function for this, users can leverage the certificate name they have already created in the database. However, certain conditions apply when using the Certificate functionality.
      > Only one column can be associated with the certificate at a time.
      > When creating a certificate, please ensure to use the column name 'Encrypted_Value' with the data type set as varbinary(max).
      > You can utilize one certificate for one column at a time. If you intend to use it for multiple columns simultaneously, it's not possible. However, you can apply the certificate functionality to one column, unmap that column, and then proceed to map it with a new column.

  • On this page, to add a list of keywords, first, select the project. Then, click on the plus icon, enter the keyword, choose the obfuscation function from the dropdown, and save it. The keyword will be added to the selected project.
  • To Edit a keyword, select the project name, locate the action column, and click on the edit icon.
  • To delete a keyword, select the project name, locate the action column, and click on the delete icon.
  • Note : Any columns added here will be displayed on the mapping page. These keywords are utilized for the Apply Keyword Search functionality.

How to Process :
  • Selecting the Project will list out all the Tables mapped for obfuscation.
  • It will not list tables that have no columns to be Obfuscated.
  • Select the tables to be obfuscated and click "Obfuscate" to apply obfuscation to all the columns that have been mapped for obfuscation.
  • On successful obfuscation, the checkbox will be greyed out and Status would be changed to Obfuscated along with Obfuscation timestamp.
  • In case of an error, the details will be displayed in the Error Column. Resolve the error and click "Obfuscate" again. Resolving the error might also address the issue in the client database.
  • Once the data is obfuscated and later on the data is restored in client database then you want to obfuscate the new data then simply You can click on Re-enable Obfuscation button and then click obfuscate button.
  • If any table is deleted in the client database, it can be removed from the Obfuscation tool by clicking the "Delete" button. This action will eliminate all mappings for the respective table from the Obfuscation Tool.
  • Note : Once you obfuscate the data, it cannot be reverted to the original state. Although you can obfuscate the data multiple times, retrieving the original data is not possible. To revert to the original data, you need to restore it from a backup in the database.
